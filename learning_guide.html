Trực quan hóa Luồng hoạt động & Phân tích Mã nguồn

Tài liệu học tập chuyên sâu giải thích chi tiết cấu trúc code từ Gymnasium wrappers đến các thuật toán tự viết PyTorch.

1. Tổng quan Dự án

Chào mừng bạn đến với cẩm nang học tập thực hành của dự án Pong Reinforcement Learning Comparison. Đây là một hệ thống so sánh (benchmark suite) toàn diện được thiết kế để đo lường hiệu năng huấn luyện giữa các thuật toán Reinforcement Learning.

Dự án so sánh trực tiếp hai trường phái tiếp cận cốt lõi trong học tăng cường:

Trường phái tiếp cận Thuật toán triển khai Đặc điểm nổi bật
Value-based (Dựa trên giá trị) DQN & Dueling Double DQN (D3QN) tự viết Học Off-policy, tối ưu hóa hàm giá trị Q qua Bellman update, sử dụng Replay Buffer để tái sử dụng dữ liệu.
Policy-based (Dựa trên chính sách) PPO (Tự viết & Baseline Stable-Baselines3) Học On-policy ổn định, sử dụng clipped objective để ngăn chính sách thay đổi quá đột ngột, tính toán GAE.

2. Luồng Hoạt động Trực quan

Dưới đây là sơ đồ luồng dữ liệu tương tác độc quyền được thiết kế để trực quan hóa quá trình xử lý từ khung hình Atari thô đến bước học mạng neural của Agent. Hãy click chuột vào bất kỳ bước nào trong sơ đồ để xem giải thích nghiệp vụ chi tiết.

1. Atari Frame (Khung hình gốc)
Khung hình gốc trích xuất trực tiếp từ máy giả lập Atari (ALE) có độ phân giải màu 210x160x3 RGB. Khung hình đơn lẻ này không chứa thông tin động về hướng chuyển động và tốc độ của quả bóng, do đó cần phải được tiền xử lý.

3. Phân lớp Kiến trúc Mã nguồn

Dự án được cấu trúc theo 4 phân lớp độc lập giúp việc phát triển, kiểm thử và nghiên cứu học tập trở nên mạch lạc:

1. Điểm khởi chạy và Giao diện CLI layer:cli-entry

Chứa các mã nguồn CLI điều khiển huấn luyện và công cụ so sánh trực quan kết quả:

  • scripts/run_scratch.py Điểm khởi chạy huấn luyện các agent tự phát triển từ đầu (DQN, Dueling DDQN, PPO).
  • scripts/run_sb3.py Điểm khởi chạy huấn luyện baseline sử dụng thư viện Stable-Baselines3 làm quy chiếu so sánh.
  • scripts/evaluate.py Script đánh giá, Benchmark trực quan và vẽ biểu đồ so sánh hiệu năng các mô hình.
2. Các Agent tự viết từ đầu (Custom Agents) layer:scratch-agents

Phân lớp cốt lõi chứa các mô hình học sâu và thuật toán được lập trình trực tiếp bằng PyTorch:

  • src/scratch/models.py Định nghĩa mạng Nature CNN dùng chung, đầu Dueling tách biệt và mạng Actor-Critic.
  • src/scratch/dqn_agent.py Lập trình Agent DQN tiêu chuẩn xử lý cập nhật Bellman và chính sách Epsilon-Greedy.
  • src/scratch/dueling_ddqn_agent.py Agent Dueling DDQN (D3QN) nâng cao cải thiện chất lượng học của mạng.
  • src/scratch/ppo_agent.py Lập trình Agent PPO từ đầu với bộ thu thập Rollouts và thuật toán GAE.
  • src/scratch/replay_buffer.py Bộ đệm lưu trữ trải nghiệm tối ưu hóa bộ nhớ cho off-policy training.
3. Bộ tiền xử lý & Tiện ích chung layer:common-utils

Các module tiền xử lý dữ liệu môi trường và tiện ích ghi log, thiết lập seed:

  • src/common/wrappers.py Bộ bọc xử lý AtariPreprocessing và FrameStackObservation của Gymnasium.
  • src/common/utils.py Hàm tiện ích vẽ biểu đồ so sánh, ghi log CSV và cố định seed ngẫu nhiên.

4. Bộ Phân tích Code Thông minh

Phần này phân tích sâu vào các khối code thuật toán cốt lõi. Hãy nhấp vào các Tab để khám phá chi tiết cách triển khai DQN, Dueling DDQN và PPO tự viết bằng PyTorch.

Hướng dẫn: Nhấp vào các huy hiệu tròn màu hồng i ngay trong khối code dưới đây để xem giải thích chi tiết về dòng lệnh đó.

Trong thuật toán DQN tự viết, trọng tâm nằm ở hàm cập nhật update() trích xuất minibatch từ Replay Buffer và tối ưu hóa hàm MSE Loss dựa trên phương trình Bellman.

src/scratch/dqn_agent.py (Trích đoạn hàm update) Python 
# Trích mẫu ngẫu nhiên một minibatch từ Replay Buffer
states, actions, rewards, next_states, dones = self.replay_buffer.sample(self.batch_size)

# Tính Q-value hiện tại của Agent đối với trạng thái hiện tại
q_values = self.online_net(states) 1
state_action_values = q_values.gather(1, actions.unsqueeze(1)).squeeze(1) 2

# Tính toán giá trị Q kỳ vọng từ Target Network (phương trình Bellman)
with torch.no_grad(): 3
    next_state_values = self.target_net(next_states).max(dim=1)[0]
    expected_state_action_values = rewards + self.gamma * next_state_values * (1.0 - dones) 4

# Tính toán hàm mất mát Mean Squared Error
loss = self.loss_fn(state_action_values, expected_state_action_values)

# Lan truyền ngược và tối ưu hóa mạng online
self.optimizer.zero_grad()
loss.backward()
torch.nn.utils.clip_grad_norm_(self.online_net.parameters(), max_norm=10.0) 5
self.optimizer.step()

Mạng Dueling DDQN chia kiến trúc mạng thành hai nhánh: Hàm giá trị trạng thái V(s) và Hàm lợi thế hành động A(s, a). Về phần tối ưu hóa, nó kết hợp cơ chế Double DQN để giải quyết vấn đề Overestimation (đánh giá quá cao giá trị Q).

src/scratch/dueling_ddqn_agent.py (Hàm update Double DQN) Python 
# Tính toán giá trị Q kỳ vọng bằng công thức Double DQN
with torch.no_grad():
    # 1. Dùng Online Network để lựa chọn hành động tốt nhất cho trạng thái s'
    best_actions = self.online_net(next_states).argmax(dim=1, keepdim=True) 1
    
    # 2. Dùng Target Network để định lượng giá trị Q của hành động tốt nhất đó
    next_state_values = self.target_net(next_states).gather(1, best_actions).squeeze(1) 2
        
    # 3. Tính mục tiêu Bellman
    expected_state_action_values = rewards + self.gamma * next_state_values * (1.0 - dones)
src/scratch/models.py (Cơ chế Dueling Forward) Python 
# Tính toán song song hai nhánh Value Stream V(s) và Advantage Stream A(s, a)
values = self.value_stream(flat_out) 3
advantages = self.advantage_stream(flat_out)

# Kết hợp: Q(s, a) = V(s) + A(s, a) - mean(A(s, a'))
q_values = values + (advantages - advantages.mean(dim=1, keepdim=True)) 4
return q_values

Trong PPO, mục tiêu huấn luyện là Clipped Policy Loss và Value Loss kết hợp với hệ số Entropy để khuyến khích khám phá môi trường.

src/scratch/ppo_agent.py (Hàm update PPO Joint Loss) Python 
# Tính tỷ lệ xác suất chính sách mới / cũ: r_t(theta)
ratios = torch.exp(new_log_probs - b_old_log_probs) 1

# Tính toán hai nhánh của Clipped Policy Loss
surr1 = ratios * b_advantages
surr2 = torch.clamp(ratios, 1.0 - self.clip_ratio, 1.0 + self.clip_ratio) * b_advantages 2
policy_loss = -torch.min(surr1, surr2).mean() 3

# Tính toán hàm mất mát giá trị Value Loss (Clipped)
v_loss_unclipped = (v_pred - b_returns) ** 2
v_pred_clipped = b_old_values + torch.clamp(v_pred - b_old_values, -self.clip_ratio, self.clip_ratio)
v_loss_clipped = (v_pred_clipped - b_returns) ** 2
value_loss = 0.5 * torch.max(v_loss_unclipped, v_loss_clipped).mean() 4

# Tính toán tổng hàm mất mát (Joint Objective Loss)
loss = policy_loss + self.val_coef * value_loss - self.entropy_coef * entropy 5

5. Xử lý Lỗi & Tối ưu hóa Thực tế

Reinforcement Learning là một lĩnh vực rất khó debug vì hành vi của Agent có tính ngẫu nhiên. Dưới đây là các cơ chế xử lý lỗi và tối ưu được áp dụng trong dự án:

Cực kỳ quan trọng: Tránh tràn bộ nhớ RAM (RAM Overflow)
Một trạng thái stack chứa 4 khung hình 84x84. Nếu lưu trữ mặc định kiểu float32 (4 bytes/pixel), 100,000 trạng thái trong Replay Buffer sẽ chiếm tới ~11.28 GB RAM. Bằng cách ép kiểu dữ liệu về np.uint8 (1 byte/pixel) trong replay_buffer.py, lượng RAM tiêu thụ giảm 4 lần, chỉ còn ~2.82 GB RAM. Khi trích mẫu (sampling), ta mới thực hiện chia cho 255.0 để đưa về miền [0.0, 1.0].
Xử lý Tương thích Gymnasium (Terminated & Truncated)
Sự thay đổi API từ phiên bản Gymnasium v0.26+ khiến nhiều bài toán RL cũ bị crash. File ppo_agent.py tự động nhận diện số lượng tham số trả về từ env.step() để tương thích ngược, đảm bảo lấy đúng trạng thái kết thúc (done = terminated or truncated) when chạy trên các máy tính có cài đặt thư viện khác nhau.

6. Chạy Thực nghiệm & Benchmark

Để khởi chạy huấn luyện và tiến hành so sánh kết quả giữa các agent, bạn cần sử dụng môi trường uv đã được cấu hình sẵn trong dự án hiện tại.

1. Huấn luyện Agent tự viết (Scratch Agent)

Bạn có thể lựa chọn huấn luyện dqn, dueling_ddqn, hoặc ppo:

uv run scripts/run_scratch.py --agent dueling_ddqn --episodes 100 --lr 1e-4

2. Huấn luyện Baseline Agent (Stable-Baselines3)

Huấn luyện PPO baseline chuẩn của thư viện SB3:

uv run scripts/run_sb3.py --algo ppo --timesteps 50000

3. Đánh giá và Vẽ biểu đồ so sánh

Sau khi lưu kết quả log CSV dưới thư mục data/logs/, chạy lệnh sau để tự động tổng hợp kết quả huấn luyện và vẽ biểu đồ so sánh phần thưởng nhận được:

uv run scripts/evaluate.py

7. Câu hỏi Ôn tập Củng cố

Hãy tự kiểm tra kiến thức lập trình Reinforcement Learning của bạn với các câu hỏi tương tác dưới đây:

Câu hỏi 1: Tại sao Double DQN lại khắc phục được vấn đề Overestimation của DQN gốc?
A. Nó nhân đôi kích thước của bộ đệm Replay Buffer để học hiệu quả hơn.
B. Nó tách biệt việc chọn hành động tốt nhất (bằng Online Net) và việc định lượng giá trị Q của hành động đó (bằng Target Net).
C. Nó tăng hệ số chiết khấu gamma lên cao để ưu tiên phần thưởng dài hạn.
Câu hỏi 2: Vai trò chính của GAE (Generalized Advantage Estimation) trong PPO là gì?
A. Nhằm cân bằng giữa độ chệch (bias) và phương sai (variance) khi ước lượng giá trị Advantage giúp tối ưu hóa policy ổn định.
B. Nhằm đẩy nhanh tốc độ lấy mẫu từ môi trường song song.
C. Nhằm mục đích tự động cắt giảm kích thước ảnh trạng thái đầu vào.
